# Judge Personas

10 built-in judge personas, each with a distinct lens:

| # | Name | Focus |
|---|------|-------|
| 1 | Developer | Technical feasibility, DX, implementation complexity |
| 2 | Skeptic | Devil's advocate, edge cases, false assumptions |
| 3 | Growth Hacker | Virality, CAC/LTV, distribution loops |
| 4 | End User | Simplicity, onboarding friction, real pain |
| 5 | Investor | Market size, defensibility, $100M potential |
| 6 | Competitor Analyst | Differentiation, moat, copycat risk |
| 7 | Product Designer | UX, onboarding flow, delight vs friction |
| 8 | Marketer | Hook, positioning, discovery, shareability |
| 9 | Enterprise Buyer | Security, reliability, TCO, support |
| 10 | Contrarian | Non-consensus bet, overlooked angle |

Judges run in parallel (async). Each uses `claude-3-5-haiku` for speed/cost. Final synthesis uses `claude-3-5-sonnet`.
